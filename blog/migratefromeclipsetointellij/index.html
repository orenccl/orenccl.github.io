轉移Eclipse舊專案到IntelliJ的心得

By 英俊善良 2023-08-19 Ide

公司請我參與一個 Vue 前端專案，接手後才發現是使用 Eclipse 建置的 Maven Spring Boot 全端專案。

當下其實滿傻眼的，因為我沒接觸過 Java 跟該語言的周邊生態，也沒有後端工作經驗，公司說法也只是 Vue 前端。

身為前端工程師，全身心都已經是 VS Code 的形狀了，叫我用 Eclipse 開發根本是酷刑!

馬上 Google 轉移 Eclipse 專案到其他編輯器的方法，就找到了 IntelliJ 跟 VS Code ，當時內心一陣吶喊，太棒拉!

但實際操作過後，個人認為 VS Code 在 Java 專案開發上的支援還不夠完善，除了需要裝 Java Extension Pack、Spring Boot Enxtension Pack ，還需要加裝 Community Server 使專案能夠在 Tomcat 上運行，並且需要自己全手動去調 Run Configuration 來讓程式順利運行。

這對一個不熟悉 Java 周邊生態的人來說非常困難且繁瑣，因此妥協，過了一陣子使用 VS Code 編輯完再回到 Eclipse 去 Debug、Compile 的日子。

但日子過久了最終還是忍不住，只好把希望轉向要付費的 IntelliJ ，覺得一個月 500 也沒差，趕快結束我的痛苦吧!

沒想到實際使用驚為天人，只要載入 Projec t 時，選取 Maven 的 pom.xml ，IntelliJ 就能自動載入，且設定 Run Configutation 時也有相對應的提示。

編輯器本身也相當現代化，花少許時間習慣就能上手，而且非常萬幸，在我使用 IntelliJ 沒多久，公司就傳出要購買授權來給員工使用，希望是真的!

大家如果在轉移到 IntelliJ 的過程中遇到問題的話可以在底下留言聯絡我，也許我能給予一些幫助~
